# Foot Base Mesh Generator

MシリーズMac上で動作する、造形用の足ベースメッシュをパラメトリックに生成するPythonアプリです。Webcamは使用せず、PySimpleGUIのスライダー操作からOpenCVの棒状2Dプレビューを更新し、OBJ形式で書き出します。

## セットアップ

Python 3環境を用意し、必要ライブラリをインストールします。

```bash
python3 -m pip install -r requirements.txt
```

## 起動方法

Finderからアプリ風に起動する場合は、以下のどちらかをダブルクリックします。

- `FootBaseMeshGenerator.app`: Terminal経由で起動する簡易Macアプリ。起動ログやエラーが見えます。
- `FootBaseMeshGenerator.command`: 初回セットアップ込みの起動用。`.venv` を作成し、必要ライブラリを入れてから起動します。アプリ終了後は起動に使ったTerminalウィンドウを自動で閉じます。

既存の `miniforge3/envs/hand-obj` または `miniforge3/envs/pose311` に必要ライブラリがある場合は、それを自動で優先して使います。

macOSのTk起動クラッシュやLaunchServices経由のPython権限問題を避けるため、`.app` はTerminalで `.command` を起動します。

ターミナルから直接起動する場合は以下です。

```bash
python3 - <<'PY'
from main import main

main()
PY
```

依存関係の不足で `.app` 起動に失敗する場合は、先に `FootBaseMeshGenerator.command` を一度実行してください。

## 操作

- スライダーを動かすと、OpenCVで描画した足の簡易スケルトンがリアルタイムに変化します。
- `右足 / 左足` でX方向ミラーを切り替えます。
- `上面プレビュー / 側面プレビュー / 両方表示` で表示モードを切り替えます。
- `OBJ作成・保存` で現在のパラメータからメッシュを生成し、OBJファイルを書き出します。
- 保存先はアプリ直下の `output/` フォルダです。初回保存時に自動作成され、メインウィンドウ下部にも表示されます。
- 保存ファイル名は `foot_base_mesh_YYYYMMDD_HHMMSS.obj` 形式です。タイムスタンプを付けるため、既存ファイルを上書きしません。
- アプリ終了時とOBJ保存時に、現在のスライダー値を `last_preset.json` へ保存します。次回起動時は前回の足型、左右、プレビュー表示を復元します。
- `リセット` で標準状態に戻します。

## パラメータ

- 足長: かかとから指先方向までの全体長。
- 足幅: 母趾球から小趾球付近の最大幅。
- 甲の高さ: 足本体上面と足首接続部の高さ。
- かかとの幅: かかと周辺の横幅。
- かかとの大きさ: かかと球体、後足部、アキレス腱付け根まわりのボリューム。
- 土踏まずの高さ: 側面ビューと足裏側メッシュのアーチ量。
- 指の長さ: 親指以外の基準長。
- 親指の長さ: 親指だけの個別長。
- 指の太さ: 足指ブロックの太さ。
- 球体の大きさ: 足指関節と指先の球体ジョイントの大きさ。
- くるぶしの大きさ: くるぶし球体と周辺接続部の大きさ。
- 甲パーツ厚み: 甲側の中足骨、腱、指根元への埋め面の厚み。
- 指の開き: 5本指を扇状に広げる角度。
- 指の曲げ: 指セグメントの屈曲。
- 指の反り: 側面方向の持ち上がり。
- 親指角度: 親指だけの横方向角度。
- 足首角度: 足首点の傾き。
- くるぶし回転: 母趾球/小趾球付近を支点に、かかと・甲・足首側を回転させるつま先立ち用の角度。

メッシュ解像度はGUIでは調整しません。Nomad SculptやZBrush側でSDiv、Dynamesh、Voxel Remeshする前提の軽めの固定解像度で出力します。

## プリセット

- 標準: 一般的な足型。
- 幅広: 足幅と指の開きを大きめにした足型。
- 甲高: 甲の高さと足首接続部を高めにした足型。
- 扁平足気味: 土踏まずを低めにした平坦寄りの足型。
- 指長め: 足指全体を長めにした足型。
- エジプト型: 親指が最も長く、人差し指以降が徐々に短い足型。
- ギリシャ型: 人差し指が最も長く、親指が少し短い足型。
- スクエア型: 親指から中指あたりまでの長さが近い足型。

## OBJ利用想定

出力OBJは、ZBrush、Nomad Sculpt、Blenderなどへ読み込み、Dynamesh、Voxel Remesh、ZRemesher、Sculptなどで後加工するためのベースメッシュです。最初から解剖学的に完全な一体モデルにせず、骨と腱の流れを意識できる四角ブロック、関節球、5本指の軽い重なりを許容して、造形しやすい形を優先しています。

OBJ出力時は内部mm相当値を `0.004` 倍して書き出します。参考にした手ベースメッシュOBJの約1単位スケールに近づけ、Nomad SculptやZBrush上で並べて扱いやすくするためです。

Nomad SculptやZBrush側で一括SDivしやすいように、OBJは `foot_base_mesh` の単一オブジェクト/単一グループとして出力します。Nomad Sculptがマテリアル単位でオブジェクト分割することを避けるため、MTLや `usemtl` は書き出しません。色分けはOBJの頂点カラー形式、つまり `v x y z r g b` で書き出します。

足指は四角いブロックと球体ジョイントで生成します。円柱形状は使わず、付け根・中間関節・末節関節・指先へ球体を重ねます。SDiv、Dynamesh、Voxel Remesh後に節を残しやすい構成です。

足本体は一枚板ではなく、低ポリの外皮、中足骨ブロック、甲側の厚み付きロフト面、土踏まず、母趾球/小趾球、アキレス腱まわりで構成します。くるぶし周辺は四角柱ではなく球体を使います。SDiv後に細りすぎないよう、骨・腱・指根元の埋め面はやや太めに出力し、甲パーツ厚みスライダーで追加調整できます。

## ファイル構成

- `main.py`: GUI本体、イベントループ、プレビュー更新、OBJ出力処理。
- `foot_params.py`: dataclassによるパラメータ定義。
- `foot_skeleton.py`: プレビューとメッシュ生成で共通利用するスケルトン計算。
- `foot_preview.py`: OpenCVプレビュー描画とPNGバイト列変換。
- `foot_mesh_generator.py`: 足本体、かかと、甲、足指メッシュ生成。
- `obj_exporter.py`: OBJ出力。
- `presets.py`: 各プリセット値。
